# NBME JSON Importer — Technical Specification

**Last updated:** 2026-05-13  
**Status:** Stable. All blocking bugs resolved. Psych Shelf 3–8 validated. `retrievalTag` + `reviewPearl` display support added 2026-05-13. VAL-002 (figure rendering) and VAL-003 (save-valid-only) pending end-to-end validation.

---

## Overview

The NBME Gemini JSON importer (`nbme-gemini-json` pipeline) accepts a pre-structured JSON file generated by running a full NBME exam through Google Gemini with a specific extraction prompt. It bypasses OCR and PDF parsing entirely — the JSON already contains parsed stems, choices, answer keys, and explanations.

This is distinct from all other pipelines. No AI refinement step is needed (the JSON is the output of an external AI step). The import pipeline is: **validate → preview → attach figures (optional) → save**.

---

## Trigger

The importer is opened from the main library screen via a dedicated import button. The modal ID is `#modal-nbme-gemini-json-import`. The modal is opened by calling `App.openNbmeGeminiJsonImportModal()`.

## Expected JSON Schema (Input)

The JSON file must be an object with a `questions` array. Each element must have:

```json
{
  "questions": [
    {
      "questionNumber": 1,
      "stem": "A 32-year-old woman presents with...",
      "answerChoices": [
        { "letter": "A", "text": "Bipolar I disorder" },
        { "letter": "B", "text": "Major depressive disorder" },
        { "letter": "C", "text": "Borderline personality disorder" },
        { "letter": "D", "text": "Cyclothymia" },
        { "letter": "E", "text": "Dysthymia" }
      ],
      "correctAnswer": "C",
      "educationalObjective": "Borderline personality disorder is characterized by...",
      "explanationSections": [
        {
          "heading": "Correct Answer",
          "text": "Borderline personality disorder (BPD) is characterized by..."
        },
        {
          "heading": "Incorrect Answers",
          "text": "Choice A: Bipolar I disorder requires...\nChoice B: Major depressive disorder..."
        }
      ],
      "figureRefs": [
        {
          "figureId": "Figure_Q25_Lab",
          "location": "stem",
          "visibleText": [
            "Sodium: 138 mEq/L",
            "Potassium: 3.9 mEq/L",
            "Chloride: 101 mEq/L"
          ]
        }
      ]
    }
  ]
}
```

### Required fields per question:
- `questionNumber` — integer
- `stem` — string (the full clinical vignette)
- `answerChoices` — array of `{letter, text}` objects, minimum 2
- `correctAnswer` — string matching one of the choice letters

### Optional fields per question:
- `educationalObjective` — plain text educational objective
- `explanationSections` — array of `{heading, body[]}` objects
- `figureRefs` — array of figure reference objects (see Figure References below)
- `tables` — array of embedded table objects (see Tables below)
- `sharedGroup` — shared-stem/shared-choices group descriptor (see Shared Groups below)
- `retrievalTag` — short memory anchor encoding the exact tested concept (e.g., `"PTSD duration threshold"`). Stored at `q.retrievalTag` and `q.metadata.retrievalTag`. Also added to `q.tags[0]` for backward compatibility with tag-based UI surfaces.
- `reviewPearl` — one-line high-yield review statement (e.g., `"PTSD requires symptoms lasting more than 1 month after trauma."`). Stored at `q.reviewPearl` and `q.metadata.reviewPearl`.
- `extractionWarnings` — string array; merged into `q.metadata.extractionWarnings`

### Optional top-level fields:
- `sourceFormat` — string describing the input format. Recognized values: `pdf`, `docx`, `ocr_text`, `screenshot`, `mixed`, `rtf`, `unknown`. Other values produce a validation warning but do not block import.

## Internal Quiz Schema (Output)

After normalization, each question is stored in localStorage as part of a test's `questions` array with this structure:

```javascript
{
  n: 1,                        // question number (integer)
  t: "A 32-year-old woman...", // full stem text (from q.stem)
  o: [                         // answer choices
    { l: "A", t: "Bipolar I disorder" },
    { l: "B", t: "Major depressive disorder" },
    // ...
  ],
  c: "C",                      // correct answer letter
  e: {                         // per-choice explanations (keyed by letter)
    "A": "Bipolar I disorder requires...",
    "B": "Major depressive disorder..."
  },
  tags: ["PTSD duration threshold"],  // retrievalTag promoted to tags[0], or [] if absent
  retrievalTag: "PTSD duration threshold",   // top-level, '' if not in input
  reviewPearl: "PTSD requires symptoms lasting more than 1 month after trauma.",  // top-level, '' if not in input
  educationalObjective: "Borderline personality disorder is characterized by...",
  correctBlurb: "<strong>Correct Answer</strong><br><br>BPD is characterized by...",
  metadata: {
    sourceType: "nbme-gemini-json",
    questionNumber: 1,
    originalStem: "A 32-year-old woman...",
    retrievalTag: "PTSD duration threshold",  // mirrored in metadata for lookup compatibility
    reviewPearl: "PTSD requires symptoms lasting more than 1 month after trauma.",
    figureRefs: [ /* copied from input */ ],
    figureAttachments: {
      "Figure_Q25_Lab": "data:image/png;base64,..."  // only if user attached
    },
    schemaVersion: "nbme-gemini-json-v1",
    importedAt: "2026-05-12T..."
  }
}
```

### Key field notes:
- `q.t` — the primary stem field used by the quiz renderer. Must be the full untruncated stem.
- `q.correctBlurb` — pre-escaped HTML built from `explanationSections[0]` (the "Correct Answer" section). Set as `innerHTML` by `buildExplanationHTML`. Built by `_ngjBuildCorrectBlurb()`.
- `q.e` — per-choice explanations parsed from `explanationSections[1]` (the "Incorrect Answers" section). Built by `_ngjBuildPerChoiceExplanations()`.
- `q.educationalObjective` — plain text, set as `textContent` by `buildExplanationHTML`.
- `q.retrievalTag` — top-level string. Read by `getRetrievalTag(q)`, which also checks `q.metadata.retrievalTag` as fallback.
- `q.reviewPearl` — top-level string. Read by `getReviewPearl(q)`, which also checks `q.metadata.reviewPearl` as fallback.
- `q.metadata.figureAttachments` — base64 DataURLs for user-uploaded figure images, keyed by `figureId`. This is what enables inline image rendering when the quiz displays `[FIGURE: figureId]` markers.

## Figure References

`figureRefs` in the input JSON describe figures, tables, or lab values that appear visually in the original exam but cannot be embedded in plain text.

### `figureRef` schema:
```json
{
  "figureId": "Figure_Q25_Lab",    // unique identifier for this figure within the question
  "location": "stem",              // where the figure appears: "stem", "choices", etc.
  "visibleText": [                 // optional: text rows from the figure (lab values, table cells)
    "Sodium: 138 mEq/L",
    "Potassium: 3.9 mEq/L"
  ]
}
```

### How figure markers work in the stem:
When Gemini processes the original exam, it replaces inline figures with `[FIGURE: figureId]` markers in the stem text. For example:
```
"stem": "A 25-year-old man's lab results are shown. [FIGURE: Figure_Q25_Lab] Which of the following..."
```

### How the app renders them:
1. The stem is rendered as HTML by `window.buildStemHTML(q.t)`.
2. `window.buildQuestionStemHTML(q, highlightedStemHtml)` then calls `window._replaceFigureMarkersInStemHtml(html, q)`.
3. `_replaceFigureMarkersInStemHtml` uses a regex to find all `[FIGURE: figureId]` patterns and replaces each with the result of `window._ngjFigureToHTML(figureId, q)`.
4. `_ngjFigureToHTML(figureId, q)` checks in this order:
   - If `q.metadata.figureAttachments[figureId]` exists → renders `<img src="data:...">` inline
   - Else if `figureRef.visibleText` is a non-empty array → renders a styled lab-values table
   - Else → renders a placeholder box: `📷 Figure not attached: figureId (location: stem)`

### Figure attachment workflow:
After JSON is parsed and validated, if any questions have `figureRefs`, the import modal shows a "Figure Attachment" panel. Each figureRef gets a row with:
- figureId and location
- An "Attach image" file input (accepts `image/*`, max 2.5MB)
- After attachment: a thumbnail and "Remove" button

The base64 DataURL is stored in `_nbmeGeminiJsonImport.figureAttachments[figureId]`. When saving, `createTestFromNbmeGeminiJsonImport` copies the relevant attachments into each question's `q.metadata.figureAttachments`.

## Tables

The `tables` field carries embedded structured tables (lab values, vitals panels, etc.) that were extracted from the original exam into structured form rather than as `figureRef.visibleText`.

### `tables[]` schema:
```json
{
  "title": "Laboratory studies",
  "headers": ["Test", "Result"],
  "rows": [
    ["Sodium", "138 mEq/L"],
    ["Potassium", "3.9 mEq/L"]
  ]
}
```

Tables are stored in `q.metadata.tables`. They are NOT automatically rendered inline in the stem — they are preserved in metadata for future rendering support. (Unlike `figureRef.visibleText`, which is rendered as a lab-values table whenever a `[FIGURE: figureId]` marker is present in the stem.)

Validated in: Psych_Shelf_3 Q10 (8-row lab table), Psych_Shelf_5, Psych_Shelf_6, Psych_Shelf_7.

## Shared Groups

Some NBME questions share a common patient vignette across multiple consecutive questions (shared stem) or share a common answer choice list (shared choices). These are encoded in the `sharedGroup` field.

### `sharedGroup` schema:
```json
{
  "id": "group_q33_q36",
  "questionRange": { "start": 33, "end": 36 },
  "linkedQuestionIds": [33, 34, 35, 36],
  "sharedStem": "A 45-year-old man is brought to the emergency department by his wife...",
  "sharedInstruction": "For each patient description, select the most likely diagnosis.",
  "sharedChoices": [
    { "label": "A", "text": "Alcohol withdrawal" },
    { "label": "B", "text": "Benzodiazepine intoxication" }
  ]
}
```

All fields are optional. The most common form in NBME JSON files is `sharedStem` only (shared vignette, per-question answer choices).

### How shared groups are rendered:

The `sharedGroup` is stored in `q.metadata.sharedGroup`. The quiz renderer detects it via `q.metadata.sharedGroup` and uses `window.buildSharedGroupHTML(q)` to render the shared vignette above the per-question stem.

If `sharedGroup.sharedChoices` is a non-empty array with ≥2 items, it overrides `q.o` (the per-question answer choices) at quiz render time.

Validated in: Psych_Shelf_3 Q33–Q36 (shared stem, no shared choices), Psych_Shelf_4 (4 shared-stem groups).

## Validation Rules

`validateNbmeGeminiJsonImport(payload)` checks each question and returns a result with:

### Blocking errors (prevent save):
- Not a valid object
- `questions` array missing or empty
- Question missing `stem`
- Question missing `answerChoices` or fewer than 2 choices
- Question missing `correctAnswer`
- `correctAnswer` letter not found in `answerChoices`

### Warnings (allow save, flagged):
- `educationalObjective` missing
- `explanationSections` missing or empty
- `figureRefs` present but `visibleText` is empty (figure will show placeholder)
- `questionNumber` missing (will be auto-assigned from index)

### `questionResults` array:
Each entry: `{ questionNumber, status: 'ok'|'warning'|'error', errors[], warnings[] }`

### `counts` object:
`{ total, ok, withWarnings, withErrors, blocking }`

## UI Artifact Sanitizer

Screenshot-OCR-based NBME source PDFs sometimes leak navigation-bar text into stems or explanation body paragraphs. The most common form is the NBME footer bar:

```
Previous  Next  Score Report  Lab Values  Calculator  Help  Pause
```

or truncated versions such as `Lab Values Calculator` appearing inline after legitimate explanation text.

### Sanitizer functions (`index.html`)

| Function | Purpose |
|---|---|
| `window._ngjSanitizeUiArtifactsText(text, fieldPath, log)` | Sanitizes a single string; appends removal notices to `log[]` |
| `window._ngjSanitizeQuestion(q, log)` | Applies sanitizer to all text fields of a raw JSON question object |

### When it runs

`normalizeNbmeGeminiJsonImport()` calls `_ngjSanitizeQuestion(q, sanitizerWarnings)` as the **first step** before extracting any fields. Cleaned text flows into `normalizedSections`, `correctBlurb`, and per-choice explanations. Any removal notices are appended to the question's `extractionWarnings` in `metadata`.

### Fields sanitized

- `q.stem`
- `q.educationalObjective`
- `q.answerChoices[].text`
- `q.explanationSections[].heading`
- `q.explanationSections[].body[]`
- `q.figureRefs[].visibleText[]`
- `q.tables[].title`, `.headers[]`, `.rows[][]` / `.rows[].value`

Fields **not** sanitized: `correctAnswer`, labels, IDs, `retrievalTag`, metadata keys.

### Removal rules (conservative)

**Phase A — UI footer clusters:**

1. **Multi-term clusters** — two or more UI terms adjacent (any whitespace between): always removed.  
   Recognized terms: `Previous`, `Next`, `Score Report`, `Lab Values`, `Calculator`, `Help`, `Pause`

2. **Single UI term alone on a line** — a line whose entire content (after stripping whitespace) is exactly one UI term: removed.

3. **Short known artifact as terminal suffix** — `Lab Values` or `Score Report` appearing after sentence-ending punctuation at the very end of a line or string: removed, punctuation preserved.

**Phase B — OCR separator / header artifacts:**

4. **Inline separator run** — 3+ space-separated groups of only separator characters (`-–—·•|\/\`), optionally preceded by a stray `. ` (OCR dot before dashes). Replaced with a single space (not empty, to avoid fusing adjacent words).

5. **`.... Mark` bookmark** — 2+ consecutive dots followed by `Mark` (not `marked`/`marker`): removed.

6. **Period-preceded standalone `Mark`** — residual `Mark` immediately following a sentence period after an inline run was removed (e.g. `. Mark [text]`): removed, period preserved.

7. **Junk line** — a whole line (after `\n` split) is dropped if:
   - Empty/whitespace only
   - Contains `Please Walt`
   - Contains `https://t.me/`
   - After stripping all separator/punctuation chars, only `Mark` (+ ≤2 stray OCR chars) remain
   - ≥8 non-space chars with >70% being separator/punctuation characters

### Protected medical phrases

The sanitizer is **not** a word-level blocklist. Rules require adjacency, isolation, or heavy punctuation density:

| Phrase | Why safe |
|---|---|
| `next step`, `next best step` | `step` is not a UI term; no cluster forms |
| `next menstrual cycle`, `next follow-up` | same |
| `previous episode`, `previous examination` | `episode`/`examination` not UI terms |
| `help diagnose`, `can help determine` | `diagnose` not a UI term |
| `the lab values were abnormal` | `lab values` in sentence context |
| `score reported as` | `reported` breaks the two-word match |
| `marked psychomotor slowing` | starts with `marked` (not bare `Mark`) |
| `biomarker testing` | `Mark` preceded by `bio` (letter) |

**Psych_Shelf_5** (50 questions): 50 `"Lab Values Calculator"` instances removed, 0 false positives.

**Psych_Shelf_3** (50 questions): 49 fields cleaned (inline sep runs + `.... Mark` bookmarks), 0 false positives, 0 residual artifacts. Sample removals:

## Per-Choice Explanation Parsing

The "Incorrect Answers" section of `explanationSections` typically looks like:
```
Choice A: Bipolar I disorder requires at least one manic episode...
Choices B and D: These conditions share overlapping features but differ in...
Choice C: [Correct answer, not repeated here]
Choice E: Dysthymia is a milder but chronic form of depression...
```

`_ngjBuildPerChoiceExplanations(sections, answerChoices)` handles:
- Single letter: `Choice A:` or `Choice B:`
- Multi-letter: `Choices A, C, and D:` or `Choices B and E:`
- Maps each segment to the correct letter(s)
- Returns `{ A: "text", B: "text", ... }` (skips the correct answer letter)

## Explanation Rendering

Both `buildExplanationHTML` functions (local in Quiz IIFE ~line 5644, global `window.buildExplanationHTML` ~line 5819) render in this order:

1. **Educational Objective** — if `q.educationalObjective` exists: blue-bordered box, `textContent` assignment
2. **Correct Answer Blurb** — if `q.correctBlurb` exists: `innerHTML` assignment (pre-escaped HTML from `_ngjBuildCorrectBlurb`)
3. **Legacy explanation** — if `q.explanation` exists: plain text (for PDF-OCR-imported questions)
4. **Per-choice explanations** — if `q.e` exists: one block per choice letter
